Problems We're Solving in Africa's Healthcare
📒
Lost Paper Books
Patients lose paper hospital books constantly. Years of vital medical history disappears — causing repeated tests and wrong treatments.
🏃‍♂️
No Record Continuity
Moving between hospitals means starting from zero. No hospital shares records with another in Cameroon today.
📡
Poor Internet Access
Rural health workers can't access cloud systems. MediBook works offline with sync when connected.
💰
Duplicate Lab Tests
Without records, doctors order the same tests repeatedly — costing patients money they don't have.

About MediBook Africa

A complete digital health platform solving Africa's healthcare records challenge, built as a final year Computer Science degree project.

🎯 Project Purpose

MediBook Africa digitizes patient medical records, enabling secure storage and retrieval by patients and healthcare professionals. It addresses Africa's paper record challenge with AI-powered scanning, live doctor updates, and offline functionality built for low-connectivity regions.

🔬 Technology Stack
  • → Frontend: HTML5, CSS3, JavaScript (ES6+)
  • → Backend: PHP 8.x with MVC architecture
  • → Database: MySQL 8.0 with prepared statements
  • → AI: Anthropic Claude API (+ rule-based fallback)
  • → OCR: Tesseract.js for hospital book scanning
  • → PDF: TCPDF / Puppeteer for exports
  • → Offline: IndexedDB + Service Workers

Documentation

📦 Installation
  • 1. Clone project → /var/www/html/medibook
  • 2. Import database/medibook.sql to MySQL
  • 3. Copy config/config.example.php → config.php
  • 4. Update DB credentials in config.php
  • 5. Install Tesseract OCR engine for scan feature
  • 6. Run: composer install
  • 7. Visit http://localhost/medibook
🗄️ Database Tables
  • → users — Authentication for all roles
  • → patients — Extended patient profiles + health passport
  • → medical_records — All record types with source tracking
  • → scanned_records — OCR results with confidence scores
  • → vitals — Time-series vital signs
  • → live_updates — Real-time doctor → patient feed
  • → prescriptions — Medication prescriptions
  • → lab_results — Laboratory test results
  • → appointments — Scheduled appointments
  • → notifications — Multi-channel notifications (app/SMS/WhatsApp)
  • → ai_chat_history — Chatbot conversations
  • → activity_logs — Full audit trail
